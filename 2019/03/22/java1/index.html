常用设计模式

单例模式

确保某一个类只有一个实例，而且自行实例化并向整个系统提供这个实例。确保某个类只有一个对象，避免产生多个对象消耗过多的资源。

饿汉式

  • 优点
    • 线程安全
    • 实现简单
  • 缺点
    • 不使用时依然会实例化单例

懒汉式

  • 优点
    • 第一次调用getInstence才会实例化单例
  • 缺点
    • 非线程安全

静态内部类单例模式

  • 优点

    • 线程安全
    • 第一次调用getInstance方法才初始化单例

  • 缺点

    • 反序列化会重新创建对象

工厂方法模式

定义一个用于创建对象的接口，让子类决定实例化哪个类，在任何需要生成复杂对象的地方，都可以使用工厂方法模式。

常见实现

利用反射

  • 优点

    • 代码解耦
    • 具有良好拓展性(只需新增具体产品类即可)

  • 缺点

    • 代码量大，抽象层次多，只适用于复杂对象

抽象工厂模式

为创建一组相关或者是相互依赖的对象提供一个接口，而不需要制定它们的具体类，一个对象族有相同的约束可以使用抽象工厂模式。
